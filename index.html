Why you should learn Javascript

  • It's ubiquitous. It runs in all web browsers.
  • It's FAST. Don't believe what you hear, Javascript is screeching these days.
  • It's unique. Like a strange, fun mixture of OO and Functional plus a dash of insanity.
  • It's flexible and forgiving. Sometimes life is hard, Javascript understands.

Why you shouldn't learn Javascript

  • You're scared of awesome stuff
  • You think the best programming languages start with .text
  • You hate flexibility and loathe duck typing
  • You think closures and anonymous functions are for maths majors
  • It'll freaking blow your mind

Comparisons and Conditionals

So setting values is fun, but what if we need to check one variable against another? We use comparisons for this. We have a few different comparison operators. We can also perform conditional operations. Lets have a look at how that works.

Now comparisons aren't very useful by themselves. But fortunately, JavaScript has already thought of that. So we have conditionals, which use comparisons to tell if some instructions should be run.

Theres a few new things here. Notice we use if to do a conditional. The way this works is probably as you would expect. if gets checked first, if the value is true then it is executed, if not then we move on to the else if. If that is not true, we keep moving through as many else if statements as we can until we get to an else or nothing. The else is run if nothing else is true.

Objects

Objects are where things get fun in Javascript. You see, in Javascript everything is an object. Or at least everything behaves like an object. This can be convenient but also tricky.

I may have mentioned it before, not sure if you were paying attention. But everything is an object. That means you can set properties on them, throw them around, grab out their properties... do whatever you like. It's all fun :)

You may have also noticed that the dot and square bracket syntax is essentially the same. This is true. Most of the time we use the dot syntax, because it looks nice. However sometimes you need to use the square brackets, like when you are accessing numbered properties. It's really all up to style.

Functions

Functions in Javascript are nifty little things. You know how I mentioned before that everything is an object? Well thats true for functions as well. But theres a few other nice things we can do with functions. Kind of like the nice things we can do with objects.

But what does it all mean? Well it means you can create functions inline on the fly. Thats useful. It also means you can access the arguments no matter how many there are, thats also useful. Plus if you really want you can use methods on functions, which do useful things? You will see more of this.
